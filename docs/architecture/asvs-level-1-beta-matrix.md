# ASVS Level 1 / Beta Interna Matrix - BoxOps

Estado 2026-05-18. Esta matriz traduce el baseline ASVS Level 1 usado por BoxOps a areas practicas del producto y del codigo actual. Es trazabilidad local/documental para hardening beta interna; no es auditoria profesional, certificacion ASVS, pentest ni cumplimiento legal definitivo.

No desbloquea beta interna. S.8/A.1, B.4, OD.1/I.32 y, si fichaje entra en beta, F.15 siguen bloqueados mientras falten acceso real/staging, URL QA/staging, Supabase project/ref o DB URL, credenciales E2E por rol, tenant QA/staging, datos operativos controlados, SMTP real y evidencia redacted.

## Alcance

La revision usa solo codigo y documentacion disponibles en el repo:

- `PROJECT_BRIEF.md`
- `TASKS.md`
- `docs/architecture/security-baseline.md`
- `docs/architecture/tenant-rls-negative-test-matrix.md`
- `docs/operations/beta-operational-readiness-runbook.md`
- `docs/operations/tenant-readiness-checklist.md`
- `docs/operations/daily-operations-beta-readiness-runbook.md`

- `docs/product/webapp-completion-roadmap.md`
- helpers, Server Actions, rutas y migraciones existentes bajo `src/` y `supabase/migrations/`

No ejecuta validaciones reales, smokes autenticados, invitaciones, resets, purgas reales, pruebas sobre bucket real ni comprobaciones de dashboard Supabase/Vercel. No crea codigo, UI, migraciones, seeds, permisos por centro, grants UI, billing, onboarding guiado, documentos firmables, subida documental visible, payroll, IA, geolocalizacion, app nativa, push, service worker ni CacheStorage.

## Estados

| Estado | Significado |
|---|---|
| `cumple-local` | Hay evidencia local/documental suficiente para el corte beta documental, sin prometer que el entorno real ya esta validado. |
| `parcial` | Hay controles implementados o documentados, pero faltan pruebas negativas amplias, evidencia real, configuracion de entorno o cierre de deuda. |
| `bloqueado-por-entorno` | No puede cerrarse desde este workspace sin acceso real/staging, credenciales, tenant, datos o servicios externos. |
| `no-aplica-beta` | Queda explicitamente fuera de beta interna y no debe abrirse como requisito de este corte. |
| `futuro` | Candidato posterior a beta/v1 con task propia, threat model, permisos, RLS, pruebas y documentacion especifica. |

## Decision

La matriz deja S.9 mas trazable y `docs/architecture/tenant-rls-negative-test-matrix.md` convierte la deuda de pruebas negativas en inventario unico por superficie critica, pero el resultado global sigue siendo `parcial` y `bloqueado-por-entorno` para beta interna real. BoxOps mantiene una base local razonable para tenant safety, RLS, roles, Storage privado, no-geolocalizacion web y exclusiones de producto, pero no puede declararse beta lista, produccion lista, ASVS conforme ni legalmente conforme sin ejecutar los gates reales con evidencia redacted.
